每一个 ponytail 的有意捷径都用 `ponytail:` 注释标记，写明它的上限和升级路径。本技能把它们收集成一个账本，让延迟项不会悄悄变成永久。

## 扫描

grep 仓库中的注释标记，跳过 `node_modules`、`.git`、构建产物：

`grep -rnE '(#|//) ?ponytail:' .`  （如果你的技术栈用其他注释前缀，加上）

每个命中一行账本。注释前缀把只是顺带提到这个约定的散文排除在账本之外。

## 输出

每个标记一行，按文件分组：

`<文件>:<行号>, <简化了什么>. 上限: <注释中写的限制>. 升级: <重新审视的触发条件>.`

约定是 `ponytail: <上限>, <升级路径>`，所以上限和触发条件直接从注释里取。想要每行再加个 owner？加 `git blame -L<行号>,<行号>`。

标记烂掉的风险：任何 `ponytail:` 注释如果不写升级路径或触发条件，加一个 `no-trigger` 标签 —— 这些就是会悄悄烂掉的那些。

以 `<N> 个标记，<M> 个无触发条件。` 结尾。啥也没找到：`无 ponytail: 债。账本干净。`

## 边界

只读和报告，不改动任何东西。要持久化，请用户确认后写到文件（例如 `PONYTAIL-DEBT.md`）。一次性。"stop ponytail-debt" 或 "正常模式" 还原。
